Accessibility Best Practices Demo

This page demonstrates the four POUR principles from WCAG 2.0:

  • Perceivable: Information is available through multiple senses
  • Operable: Users can interact via keyboard and mouse
  • Understandable: Content is clear and predictable
  • Robust: Works with assistive technologies

1. Proper Heading Hierarchy

Headings use proper levels (h1 → h2 → h3) to create document structure. Only one h1 per page. Screen readers rely on this hierarchy for navigation.

1.1 Why Heading Order Matters

Users with screen readers can jump between headings. Skipping levels confuses navigation. Use headings for structure, not style.

1.2 Font Size Using rem Units

All font sizes use rem units (1rem = 16px by default). This respects user browser font-size settings, improving accessibility for users who enlarge text. The html { font-size: 62.5%; } trick makes math easy: 1.6rem = 16px, 2.4rem = 24px.

5. Color Contrast (WCAG AA Level)

5.1 Contrast Ratios

  • Natural text: This dark text (#333) on white has a contrast ratio of 12.6:1 (exceeds AAA, even AAA standards).
  • Links: Link text (#0066cc) has 8.6:1 contrast, meeting AA standards (4.5:1 required).
WCAG Standards:
  • AA: 4.5:1 for normal text, 3:1 for large text (≥24px)
  • AAA: 7:1 for normal text, 4.5:1 for large text

7. ARIA Live Regions for Dynamic Content

Live regions announce content changes to screen readers without moving focus. This example uses aria-live="polite" for status messages.

Ready for input
ARIA Live Properties:
  • aria-live="polite": Announce when convenient
  • aria-live="assertive": Announce immediately (urgent)
  • aria-atomic="true": Read entire region, not just changes
  • role="status": Semantic role for status updates

8. Custom Interactive Widget with ARIA

When native HTML lacks a component, use ARIA to make custom widgets accessible. This toggle switch uses role="switch" and aria-checked.

8.1 Custom Toggle Switch

Click the toggle or press Enter/Space to activate.

Accessibility Features:
  • role="switch": Identifies as a toggle control
  • aria-checked: Indicates current state (true/false)
  • aria-label: Provides accessible name
  • tabindex="0": Makes it keyboard-focusable
  • Keyboard support: Enter/Space keys trigger the action
